# Claude Code Architecture Guide: Building a Multi-User Proxy Server

This guide provides a comprehensive architecture for creating a Node.js proxy server that allows multiple users to share a single Anthropic OAuth token. It perfectly mirrors Claude Code's native logic by parsing your `.credentials.json` directly, handling the native OAuth token refresh process dynamically, routing traffic per-user, and storing usage logs to dedicated directories.

You can configure your friends' CLIs to point to your proxy by having them run:

```bash
export ANTHROPIC_BASE_URL="http://your-proxy-domain.com"
export ANTHROPIC_API_KEY="Friend-1-Secret-Key"
claude
```

Because `claude` detects an external API key (`ANTHROPIC_API_KEY`), it automatically **disables native OAuth refresh** and passes that key inside the `x-api-key` header to your server.

---

## 1. Setup & Configuration Files

Start by initializing the proxy environment.

```bash
mkdir claude-code-proxy && cd claude-code-proxy
npm init -y
npm install express http-proxy-middleware cors axios
```

### A. Your Auth Files (`config.json` & `.credentials.json`)

You will migrate your existing Claude CLI token to the proxy. Place your genuine `~/.claude/.credentials.json` directly next to your Node application:

**`.credentials.json`** (Generated by your initial `claude login`):

```json
{
  "accessToken": "sk-ant-oauth-...",
  "refreshToken": "...",
  "expiresAt": 1718000000000,
  "scopes": []
}
```

Create a **`config.json`** to manage your proxy users and define your Master User ID:

```json
{
  "MASTER_USER_ID": "d1a55f9a-1111-2222-3333-e98765432101",
  "USERS": {
    "Friend-1-Secret-Key": "Alice",
    "Friend-2-Secret-Key": "Bob"
  }
}
```

## 2. Server Implementation (`server.js`)

This server encapsulates user authentication, request logging (saving raw requests to folders), and dynamic OAuth token refreshing (mirrored directly from `src/services/oauth/client.ts`).

```javascript
const express = require("express");
const {
  createProxyMiddleware,
  responseInterceptor,
} = require("http-proxy-middleware");
const fs = require("fs");
const path = require("path");
const axios = require("axios");

const app = express();

// --- 1. Load Configurations ---
const config = JSON.parse(fs.readFileSync("./config.json", "utf8"));
const VALID_USERS = config.USERS;
const MASTER_USER_ID = config.MASTER_USER_ID;

// Claude Code's hardcoded Client ID for Prod (from src/constants/oauth.ts)
const CLAUDE_CLIENT_ID = "9d1c250a-e61b-44d9-88ed-5944d1962f5e";
const TOKEN_URL = "https://platform.claude.com/v1/oauth/token";

// --- 2. OAuth Token Manager ---
let activeTokens = JSON.parse(fs.readFileSync("./.credentials.json", "utf8"));

/**
 * Replicates `refreshOAuthToken` locally inside the proxy.
 */
async function ensureValidToken() {
  const now = Date.now();
  const bufferTime = 5 * 60 * 1000; // 5 minutes buffer

  // If the token expires in the next 5 mins, refresh it!
  if (now + bufferTime >= activeTokens.expiresAt) {
    console.log("Token expiring soon, attempting refresh...");
    try {
      const response = await axios.post(
        TOKEN_URL,
        {
          grant_type: "refresh_token",
          refresh_token: activeTokens.refreshToken,
          client_id: CLAUDE_CLIENT_ID,
          // Scope mirroring internal CLI array
          scope:
            "user:profile user:inference user:sessions:claude_code user:mcp_servers user:file_upload",
        },
        {
          headers: { "Content-Type": "application/json" },
        },
      );

      // Update our memory and disk storage
      activeTokens.accessToken = response.data.access_token;
      // The API might issue a new refresh token or let us keep the old one
      activeTokens.refreshToken =
        response.data.refresh_token || activeTokens.refreshToken;
      activeTokens.expiresAt = now + response.data.expires_in * 1000;

      fs.writeFileSync(
        "./.credentials.json",
        JSON.stringify(activeTokens, null, 2),
      );
      console.log("Token successfully refreshed and saved.");
    } catch (error) {
      console.error(
        "Token refresh failed:",
        error.response?.data || error.message,
      );
    }
  }
  return activeTokens.accessToken;
}

// --- 3. Directory Logging System ---
function logUserRequest(userName, requestBody) {
  const logsDir = path.join(__dirname, "logs", userName);
  if (!fs.existsSync(logsDir)) {
    fs.mkdirSync(logsDir, { recursive: true });
  }
  const filename = `req_${Date.now()}.json`;
  fs.writeFileSync(
    path.join(logsDir, filename),
    JSON.stringify(requestBody, null, 2),
  );
}

// Parse body for our logger before it hits proxy
app.use(express.json({ limit: "50mb" }));

// --- 4. User Authentication & Logging Middleware ---
app.use(async (req, res, next) => {
  // Only intercept Claude API limits
  if (req.path.startsWith("/v1/messages")) {
    const friendKey = req.headers["x-api-key"];
    const userName = VALID_USERS[friendKey];

    if (!friendKey || !userName) {
      return res.status(401).json({ error: "Unauthorized: Invalid Proxy Key" });
    }

    // Validate Anthropic Token before passing
    await ensureValidToken();

    // Log the prompt traffic to their personal folder
    logUserRequest(userName, req.body);
  }
  next();
});

// --- 5. Telemetry Sinkhole ---
app.all("/api/event_logging/batch", (req, res) => {
  res.status(200).send({ success: true });
});

// --- 6. The Anonymizing Proxy Middleware ---
const claudeProxy = createProxyMiddleware({
  target: "https://api.anthropic.com",
  changeOrigin: true,

  // We parsed the body to log it, so we must recreate the stream for the proxy
  onProxyReq: (proxyReq, req, res) => {
    // Strip the friend's fake key and inject your real OAuth Access Token
    proxyReq.setHeader("authorization", `Bearer ${activeTokens.accessToken}`);
    proxyReq.removeHeader("x-api-key");

    // Substitute User Identity (Anonymization)
    proxyReq.setHeader(
      "User-Agent",
      "claude-code/1.0.0 (Macintosh; Intel Mac OS X 10_15_7)",
    );

    // Sometimes identifying IDs are placed in custom headers. Purge them.
    proxyReq.removeHeader("anthropic-client-id"); // General safety

    // 3. The true Identity Override!
    // Claude Code natively transmits its device/user IDs inside the request JSON body
    // via a `metadata.user_id` property. We must rewrite this payload before sending it!
    if (req.body) {
      if (req.body.metadata && req.body.metadata.user_id) {
        // Completely overwrite their device_id to make it look like our Master
        req.body.metadata.user_id = JSON.stringify({
          device_id: MASTER_USER_ID,
          account_uuid: MASTER_USER_ID,
          session_id: MASTER_USER_ID,
        });
      }

      const bodyData = JSON.stringify(req.body);
      proxyReq.setHeader("Content-Type", "application/json");
      proxyReq.setHeader("Content-Length", Buffer.byteLength(bodyData));
      proxyReq.write(bodyData);
    }
  },
});

app.use("/", claudeProxy);

const PORT = 3000;
app.listen(PORT, () =>
  console.log(`Claude Code Proxy running on port ${PORT}`),
);
```

## 3. Disabling Client Telemetry

Because `claude` also pings telemetry independently of the API requests, your friends MUST turn off telemetry tracking on their machines.
Provide this setup script to them so they inject this straight into their shell (`.zshrc` or `.bashrc`):

```bash
# Point to your custom Node.js Proxy!
export ANTHROPIC_BASE_URL="http://ip-of-your-nodejs-proxy.com"
export ANTHROPIC_API_KEY="Friend-1-Secret-Key"

# 1. Block firstParty Event Logging
export CLAUDE_CODE_DISABLE_TELEMETRY=1
export OTEL_SDK_DISABLED=true

# 2. Block GrowthBook Feature Flag Identifiers
export CLAUDE_CODE_DISABLE_GROWTHBOOK=1
```

By ensuring their system halts metrics locally and utilizing your proxy's internal token swapper, Anthropic will receive a stream of requests that genuinely appear to stem exclusively from your `MASTER_USER_ID`.
